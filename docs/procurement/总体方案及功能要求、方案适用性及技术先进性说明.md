## 一、项目理解与需求分析

### 1.1 项目背景与采购本质

本项目面向**国产结构动力学求解器自主研发**，C 包采购"结构随机振动分析模块"一套。与采购商业软件授权根本不同，招标文件明确要求：**中标人交付软件模块及可编译该模块的全部源代码，自交付起软件模块及全部代码的知识产权归采购人所有**。这决定了 C 包的本质是"随机振动算法库委托研发 + 源代码与知识产权完整移交"，对投标人的要求不是"有一套能用的软件"，而是：

（1）**算法自主可控**——随机振动全部核心算法（功率谱、非平稳时域、响应谱）具备完整的数学原理依据与自主代码实现，不依赖任何受限的第三方商业库；

（2）**代码可移交、可维护**——源代码工程化、文档化，采购人接手后能够独立编译、修改、扩展与二次开发；

（3）**可嵌入自研框架**——模块接口清晰、数据协议明确，能够嵌入采购人自研的国产结构动力学求解器框架，并支持与国产软硬件平台集成。

### 1.2 C 包技术任务的本质理解

招标文件第三章对 C 包的定位是："面向国产结构动力学求解器研发，提供结构随机振动功率谱分析算法库、结构随机振动时域分析算法库和响应谱分析算法库。"我方对其技术本质的理解如下：

确定性动力分析回答"给定载荷下结构怎么动"，而**随机振动回答"在统计描述的随机激励下，结构响应的统计特征是什么"**——火箭发射段的宽频随机载荷、路面对车辆的随机激励、地震对结构的随机/反应谱激励，其输入本身无法用确定的时间函数描述，只能用**功率谱密度**或**反应谱**刻画。C 包要交付的，正是把"随机激励的统计描述"映射为"结构响应的统计描述"的三类算法：

（1）**平稳随机振动（频域）**——已知输入功率谱矩阵 $\mathbf S_{FF}(\omega)$，求响应功率谱 $\mathbf S_{uu}(\omega)$ 与响应均方根 $\sigma_u$（RMS）；核心是频响函数 $\mathbf H(\omega)$ 与谱关系 $\mathbf S_{uu}=\mathbf H\,\mathbf S_{FF}\,\mathbf H^{\mathsf H}$；

（2）**非平稳随机振动（时域）**——输入统计特征随时间变化（如发射、冲击的调制随机过程），求时变均方根 $\sigma_u(t)$ 与响应概率密度函数 $p_u(u,t)$；

（3）**响应谱分析**——已知设计反应谱（抗震、舰船冲击设计的标准输入形式），用模态叠加估计结构峰值响应 $R$（由各阶模态峰值 $r_i$ 组合而成）。

### 1.3 C 包与 A、B 包的关系及在整体求解器中的定位

本项目 A、B、C 三包面向同一个国产结构动力学求解器，共享同一套有限元建模底座（混合网格 → 单元/材料/连接/边界 → $\mathbf M$/$\mathbf C$/$\mathbf K$ 组装），并在数据上层层衔接，构成"模态 → 确定性响应 → 随机响应"的完整动力学分析链：

- **A 包（结构模态分析）**——求结构的固有频率与振型，是整个动力学分析体系的地基；
- **B 包（结构动力响应分析）**——在模态基础上求确定性载荷（瞬态/频响）下的响应，且招标文件明确要求其"支持平稳随机振动频域求解调用和非平稳随机振动时域求解调用，为后续随机振动分析模块提供稳定接口"——即 B 包在设计上即为 C 包预留了求解调用入口；
- **C 包（结构随机振动分析，本投标包）**——处理随机/反应谱激励下的统计响应，位于这条分析链的最上层：其 CQC/SRSS 功率谱与响应谱四种叠加法以**模态数据（A 包能力）**为输入，其 PEM 与非平稳时域分析以**频域/时域求解能力（B 包能力）**为后端。

也就是说，在采购人最终的整体求解器蓝图中，**C 包自然地以 A 包的模态数据与 B 包的求解引擎为上游**。我方对这一关系的应对策略分别面向"单独中 C 包"与"采购人整体集成"两种情形：

（1）**单独中 C 包时——C 模块自包含全部上游能力**，内置模态、频响、时间积分支撑引擎，确保 C 包独立可交付、可运行、可验收，不以 A、B 包是否同时采购为前提（详见 1.4）；

（2）**面向整体集成时——C 模块的数据接口与 A、B 包的自然边界保持一致**：模态数据结构对齐 A 包输出形式，频域/时域求解调用接口对齐 B 包"为随机振动预留的调用接口"，使采购人后续无论由我方或第三方补齐 A、B 模块，都能与本 C 模块零改造对接，共同组成统一的国产求解器，避免接口孤岛与重复开发。

这一设计既保证 C 包作为独立标的的完整性，又使其天然契合采购人"三模块协同、统一求解器"的总体目标——这正是对招标文件"可嵌入自研国产求解器框架"要求的深层回应。

### 1.4 C 包自包含性的关键判断

承上，在整体求解器蓝图中 C 包以 A 包的模态、B 包的求解为上游；而**单独投 C 包时，这些上游能力必须由 C 模块自身提供**——三类算法**内部依赖一条完整的结构动力学计算链**：

- 功率谱法的 **CQC / SRSS** 与**响应谱法**都需要结构的**模态**（固有频率、振型、模态参与因子）；
- **PEM（虚拟激励法）**需要在各频点求解**频率响应**；
- **非平稳时域分析**需要**时间积分**求解器。

因此，**单独投 C 包时，C 模块必须自包含一套支撑引擎**：有限元模型导入（BDF，Bulk Data File，NASTRAN 体散数据文件，主流有限元前处理软件输出的标准模型文件格式）→ 单元/材料/连接/边界解析 → 质量阵 $\mathbf M$、阻尼阵 $\mathbf C$、刚度阵 $\mathbf K$ 组装 → 内置模态求解、频率响应、时间积分 → 随机振动三类算法 → 结果导出（HDF5，Hierarchical Data Format version 5，层次化科学数据存储格式）。这是 C 模块作为独立可交付物的工程现实，也是我方总体方案的设计起点。这一点同时印证于招标文件 C 包 ▲ 指标——20 项中有 9 项是对单元、连接、质量、阻尼、材料、边界的建模支撑要求，正说明 C 模块需自带完整建模底座。

### 1.5 对 ★ 关键指标与 ▲ 重要指标的理解

招标文件明确：C 包含 **3 个 ★ 功能**（平稳随机振动、非平稳随机振动、响应谱分析），★ 为关键指标，不满足将导致响应被拒绝（废标）；含 **20 个 ▲ 性能指标**，▲ 项完全满足或正偏离方可得分（部分满足不得分），技术分 50 分全部来自 ▲ 项（20×2.5）。多数 ▲ 项要求附"算法原理描述 + 算例测试报告"或"导入/导出文件 + 截图"。

## 二、总体方案

### 2.1 总体技术路线

C 模块采用"**一底座 + 三算法库**"的总体技术路线：

**一底座**——自包含的**随机振动支撑底座**，依次完成：BDF 模型导入 → 单元/材料/连接/边界解析 → $\mathbf M$/$\mathbf C$/$\mathbf K$ 组装与约束处理 → 内置模态分析、频率响应、时间积分三类支撑求解 → HDF5 结果导出。它为上层三个算法库提供模态数据、频响数据与时域求解能力。

**三算法库**——对应招标 C 包要求的三类算法：

（1）**功率谱分析算法库**：CQC、SRSS、PEM（虚拟激励法）求响应功率谱，及响应均方根计算；

（2）**随机振动时域分析算法库**：非平稳随机振动的时变均方根与响应概率密度函数计算；

（3）**响应谱分析算法库**：单点/多点激励下 SRSS、NRL、GRP、CQC 四种模态组合的峰值响应 $R$ 估计。

技术路线的关键决策：

**（1）模态空间作为枢纽。** 底座一次性求出模态（频率、振型、模态质量/刚度/阻尼、参与因子），CQC/SRSS 功率谱、响应谱四种叠加法全部在模态空间合成，模态计算一次、多分析复用，保证效率与结果一致性。

**（2）频响计算与 PEM 深度结合。** PEM 将随机响应谱计算精确转化为一系列确定性简谐响应计算，底座的频率响应引擎既支持直接法（逐频点稀疏复矩阵分解）也支持模态法合成，为 PEM 提供高效后端。

**（3）大规模稀疏贯穿始终。** 矩阵采用稀疏存储（CSR），模态求解采用 shift-invert Block Lanczos，频响与时域积分复用稀疏分解，支撑工程规模混合网格模型。

**（4）多工况组织。** 激励谱、反应谱、约束集合以"工况（Case）"对象组织，支持单次建模、多工况批量求解与分组导出，满足招标对多工况计算组织的要求。

### 2.2 软件总体架构设计

模块采用自下而上四层架构，各层接口明确、可独立测试：

**Layer 0 —— 数据接口层。**

（1）**BDF 导入**：自研 NASTRAN BDF 解析器，支持固定/自由格式、续行与 PARAM 卡，覆盖 C 包 ▲ 点名的全部单元、材料、连接、约束卡片，导入时进行完整性校验；

（2）**HDF5 导出**：响应功率谱、均方根云图、时变统计量、概率密度、响应谱峰值等结果按层次化数据集写入 HDF5，附元数据（模型/工况/算法参数/时间戳），保证结果可追溯、可被后处理直接消费。

**Layer 1 —— 模型组装层。** 单元库（1D 杆/梁，2D 剪切板与三/四节点壳，3D 四面体/六面体/三棱柱），连接库（RBAR/RBE2 刚性、RBE3/CSPLINE 插值、CWELD/CFAST 物理连接），材料库（MAT1/MAT2/MAT8 等），质量库（CMASS/CONM 集中与一致质量），阻尼库（CVISC/CDAMP 粘性阻尼、模态阻尼、瑞利阻尼），边界与约束（SPC/SPCD/MPC）。基础运动激励经 SPCD 通道贯通到随机/反应谱分析。

**Layer 2 —— 支撑求解层（内置）。** 模态分析（shift-invert Block Lanczos）、频率响应（直接法 + 模态法）、时间积分（隐式格式），作为随机振动三类算法的支撑引擎。

**Layer 3 —— 随机振动算法层 + 应用接口层。** 即 2.1 节三个算法库；对外提供 C/C++ API 与 Python 绑定双接口，并提供命令行驱动（输入 BDF + 控制参数 → 输出 HDF5），支持脱离框架独立运行与批量回归测试。

### 2.3 与采购人自研求解器框架的集成方案

为保证"接口清晰、可嵌入自研国产求解器框架"，我方承诺：

（1）**纯库交付**：源代码 + CMake 构建脚本交付，编译产物为静态/动态库，不捆绑 GUI、不依赖商业组件；第三方依赖仅限可再分发的开源数值库，随源码一并交付并锁定版本；

（2）**两种集成深度**：浅集成——采购人框架组装好 $\mathbf M$/$\mathbf C$/$\mathbf K$ 与模态后，直接调用本模块随机振动算法层；深集成——采购人复用本模块从 BDF 导入到 HDF5 导出的完整流水线；

（3）**数据协议文档化**：矩阵格式、模型数据结构、模态数据结构、激励谱/反应谱与结果数据结构均提供正式接口文档与示例代码；

（4）**联调支持**：交付期内安排专人与采购人研发团队按指定编译环境（含国产软硬件平台）完成构建与联调验证。

### 2.4 质量保证与验证体系

（1）**三级算例验证**：解析解算例（单自由度系统白噪声响应等有闭式解的问题）→ 标准基准算例 → 工程规模算例（复杂混合网格、多点激励），每个 ▲ 算法条款至少覆盖前两级；

（2）**交叉校核**：CQC 与 PEM 在同一算例上互校，频域平稳极限与时域非平稳长时解互校，蒙特卡洛抽样作为随机响应统计量的参考解；

（3）**自动回归**：全部算例纳入自动化回归测试集，每次代码变更后全量运行；

（4）**文档交付**：算法原理手册、用户手册（中文）、接口文档、算例报告随软件一并交付。

### 2.5 小结

总体方案以"自包含底座 + 模态枢纽 + PEM/频响深度结合"组织 C 包随机振动三类算法，以"纯库交付 + 双接口 + 数据协议文档化"保证可嵌入性，以"三级算例 + 交叉校核 + 自动回归"保证质量，完整覆盖招标文件对 C 包功能、接口、交付与知识产权的全部要求。

### 3.1 ★ 核心功能逐条响应（3 项，全部完全满足）

**★1 具备结构平稳随机振动分析功能。** 响应：**完全满足，无偏离**。基于频域谱分析理论，由输入激励功率谱矩阵 $\mathbf S_{FF}(\omega)$ 经频响函数 $\mathbf H(\omega)=(\mathbf K-\omega^2\mathbf M+i\omega\mathbf C)^{-1}$ 计算响应功率谱

$$\mathbf S_{uu}(\omega)=\mathbf H(\omega)\,\mathbf S_{FF}(\omega)\,\mathbf H^{\mathsf H}(\omega)$$

提供 CQC（完全二次组合，Complete Quadratic Combination）、SRSS（平方和开方，Square Root of the Sum of Squares）、PEM（虚拟激励法，Pseudo Excitation Method）三种求解算法，输出响应功率谱 $\mathbf S_{uu}(\omega)$ 与均方根 $\sigma_u$。

**★2 具备结构非平稳随机振动分析功能。** 响应：**完全满足，无偏离**。支持均匀调制与一般演变（evolutionary）非平稳激励模型，提供时域虚拟激励/逐步积分混合算法与样本统计途径，计算时变响应均方根 $\sigma_u(t)$ 与响应概率密度函数 $p_u(u,t)$。

**★3 具备结构响应谱分析功能。** 响应：**完全满足，无偏离**。基于设计反应谱与模态数据，按模态叠加估计峰值响应 $R$，支持单点与多点激励，提供 SRSS、NRL、GRP、CQC 四种模态组合方法。

#### （二）响应功率谱算法（▲10–▲13，完全响应）

**▲10 支持 CQC 算法求解响应功率谱。** 完全响应。采用模态叠加，将响应功率谱表示为各阶模态贡献之和并**保留全部模态交叉项**。设第 $i$ 阶模态频响 $H_i(\omega)=\big(\omega_i^2-\omega^2+2i\xi_i\omega_i\omega\big)^{-1}$（单位模态质量归一化），模态（广义）力互谱 $S_{f_if_j}(\omega)=\boldsymbol\phi_i^{\mathsf T}\mathbf S_{FF}(\omega)\boldsymbol\phi_j$，则指定自由度处的响应自功率谱为

$$S_{uu}(\omega)=\sum_{i}\sum_{j}\phi_i\,\phi_j\,H_i(\omega)\,H_j^*(\omega)\,S_{f_if_j}(\omega)$$

其中 $\phi_i$ 为第 $i$ 阶振型在该自由度处的分量。该式对密集模态（密频结构）精度可靠。证明材料：算法原理描述 + 与 PEM 互校的算例。

**▲11 支持 SRSS 算法求解响应功率谱。** 完全响应。在 ▲10 模态叠加基础上**忽略模态交叉项（仅保留 $i=j$ 项）**：

$$S_{uu}(\omega)=\sum_{i}\phi_i^2\,|H_i(\omega)|^2\,S_{f_if_i}(\omega)$$

适用于固有频率分离良好的结构，计算高效，与 CQC 在频率分离场景下互校一致。

**▲12 支持 PEM 法求解响应功率谱算法。**（提供算法原理描述与算例测试报告）完全响应。**虚拟激励法（Pseudo Excitation Method）** 将激励功率谱矩阵逐频点分解（Hermitian 半正定，特征分解/Cholesky）为 $\mathbf S_{FF}(\omega)=\sum_k \mathbf s_k(\omega)\,\mathbf s_k^{\mathsf H}(\omega)$，对每个谱分量构造确定性虚拟简谐激励 $\tilde{\mathbf F}_k(\omega,t)=\mathbf s_k(\omega)\,e^{i\omega t}$，求其确定性简谐响应 $\tilde{\mathbf u}_k(\omega)=\mathbf H(\omega)\,\mathbf s_k(\omega)$，则响应功率谱

$$\mathbf S_{uu}(\omega)=\sum_k \tilde{\mathbf u}_k(\omega)\,\tilde{\mathbf u}_k^{\mathsf H}(\omega)$$

PEM **自动完整包含全部模态交叉项与多点激励相干项**——输入相关性通过谱分解保留、并在确定性响应外积中自动进入结果——将随机响应谱计算精确转化为若干（等于 $\mathbf S_{FF}(\omega)$ 的秩，通常即独立激励通道数，远小于模态数）确定性简谐响应计算，在模态数远多于激励通道数的工程问题中较传统 CQC 全展开可显著降低计算量，是大规模多点激励问题的核心算法。提供算法原理描述与算例测试报告（与 CQC/解析解对比的精度数据）。

**▲13 支持响应均方根计算。**（提供算例测试报告）完全响应。响应均方根 $\sigma_u$ 由响应功率谱沿频率数值积分得方差求得：

$$\sigma_u^2=\int_{-\infty}^{+\infty} S_{uu}(\omega)\,\mathrm d\omega$$

输出节点/单元均方根 $\sigma_u$ 云图数据；积分采用共振峰附近自适应频率网格加密保证精度。提供算例测试报告（与单自由度系统在力功率谱 $S_0$（双边白噪声）下位移方差解析解 $\sigma_u^2=\dfrac{\pi S_0}{2\xi\omega_n^3 m^2}$ 的对比，其中 $m$、$\omega_n$、$\xi$ 分别为质量、固有频率与阻尼比）。

#### （三）非平稳随机振动算法（▲14–▲15，完全响应）

**▲14 非平稳随机振动分析支持时变响应均方根计算。**（提供算例测试报告）完全响应。非平稳激励用**演变功率谱** $\mathbf G_{FF}(\omega,t)$ 描述（区别于平稳谱 $\mathbf S_{FF}(\omega)$），常用调制模型为 $\mathbf G_{FF}(\omega,t)=|A(\omega,t)|^2\,\mathbf S_{FF}^0(\omega)$，其中 $\mathbf S_{FF}^0$ 为基准平稳谱、$A(\omega,t)$ 为时变调制（包络）函数。采用**时域虚拟激励法**：将基准谱逐频点分解为 $\mathbf S_{FF}^0(\omega)=\sum_k\mathbf s_k(\omega)\,\mathbf s_k^{\mathsf H}(\omega)$，构造时变虚拟激励 $\tilde{\mathbf F}_k(\omega,t)=A(\omega,t)\,\mathbf s_k(\omega)\,e^{i\omega t}$，经内置时间积分引擎逐步求解 $\tilde{\mathbf u}_k(\omega,t)$（因结构存在记忆效应，不能将平稳公式逐时刻照搬），重构输出演变功率谱 $\mathbf G_{uu}(\omega,t)=\sum_k\tilde{\mathbf u}_k\tilde{\mathbf u}_k^{\mathsf H}$，沿频率积分得时变方差与均方根：

$$\sigma_u^2(t)=\int_{-\infty}^{+\infty}G_{uu}(\omega,t)\,\mathrm d\omega,\qquad \sigma_u(t)=\sqrt{\sigma_u^2(t)}\ (\text{零均值时})$$

提供算例测试报告（与平稳极限解、蒙特卡洛参考解对比）。

**▲15 非平稳随机振动分析支持响应概率密度函数的计算。**（提供算例测试报告）完全响应。在线性系统、高斯激励下，响应在每个固定时刻仍为高斯分布，其边缘概率密度由 $\mu_u(t)$、$\sigma_u(t)$ 直接派生（零均值时 $\mu_u=0$）：

$$p_u(x,t)=\frac{1}{\sqrt{2\pi}\,\sigma_u(t)}\exp\!\left[-\frac{(x-\mu_u(t))^2}{2\sigma_u^2(t)}\right]$$

此为高斯闭式后处理，无需另解动力学方程；非高斯激励或非线性情形则提供样本仿真统计途径求 $p_u(x,t)$（谱表示法生成激励样本系综 → 批量时域求解 → 核密度估计）。需注意：固定时刻的边缘密度不等于整个时段最大值的分布，峰值/越界概率另需极值理论。提供算例测试报告。

#### （四）响应谱分析算法（▲16–▲20，完全响应）

先由模态 $(\omega_i,\boldsymbol\phi_i)$、阻尼比 $\xi_i$、质量阵 $\mathbf M$ 与地震输入方向向量 $\mathbf r$ 计算模态参与系数 $\Gamma_i=\dfrac{\boldsymbol\phi_i^{\mathsf T}\mathbf M\mathbf r}{\boldsymbol\phi_i^{\mathsf T}\mathbf M\boldsymbol\phi_i}$；第 $i$ 阶模态对指定响应分量的峰值为 $r_i=\Gamma_i\,\phi_i\,S(\omega_i,\xi_i)$，其中 $S(\omega_i,\xi_i)$ 为设计反应谱在 $(\omega_i,\xi_i)$ 处的纵坐标、$\phi_i$ 为振型在该分量处的值（目标为加速度、内力或应力时，应取相应反应谱量并经恢复算子换算，不能照搬位移峰值式）。结构峰值响应 $R$ 由各阶峰值 $r_i$ 按以下方法组合（各阶峰值通常不在同一时刻发生，组合结果一般不保留正负号与峰值发生时刻）。各叠加法提供算法原理描述与算例测试报告。

**▲16 响应谱分析支持单点、多点激励施加。** 完全响应。支持单一支座统一激励，亦支持多支座输入不同反应谱（多点/多支座激励），按规范方法组合各支座贡献得峰值响应 $R$（多支座输入还涉及支座运动的空间相关性，按规范组合规则处理，不直接套用单一方向的 $\Gamma_i$）。

**▲17 支持 SRSS 叠加法。**（提供算法原理描述与算例测试报告）完全响应。$R=\sqrt{\sum_i r_i^2}$，假设各阶频率分离良好、互不相关。

**▲18 支持 NRL 叠加法。**（提供算法原理描述与算例测试报告）完全响应。NRL（美国海军研究实验室法，Naval Research Laboratory）取最大模态绝对值 + 其余模态 SRSS：$R=|r_{\max}|+\sqrt{\sum_{i\ne\max} r_i^2}$，为舰船设备冲击设计标准方法。

**▲19 支持 GRP 叠加法。**（提供算法原理描述与算例测试报告）完全响应。GRP（分组法，Grouping Method，由 Gupta 提出）——频率相近的模态先分组，组内绝对和、组间 SRSS，处理密频效应。

**▲20 支持 CQC 叠加法。**（提供算法原理描述与算例测试报告）完全响应。基于模态相关系数 $\rho_{ij}$ 的完全二次组合 $R=\sqrt{\sum_i\sum_j \rho_{ij}\, r_i r_j}$；相关系数由频率比 $\beta_{ij}=\omega_j/\omega_i$ 与阻尼比按常见形式计算：

$$\rho_{ij}=\frac{8\sqrt{\xi_i\xi_j}\,(\xi_i+\beta_{ij}\xi_j)\,\beta_{ij}^{3/2}}{(1-\beta_{ij}^2)^2+4\xi_i\xi_j\beta_{ij}(1+\beta_{ij}^2)+4(\xi_i^2+\xi_j^2)\beta_{ij}^2}$$

（工程实现须明确所采用规范版本、频率比方向与阻尼假设），为密频结构的标准方法。
