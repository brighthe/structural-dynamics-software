## 一、项目理解与需求分析

### 1.1 项目背景与采购本质

本项目面向**国产结构动力学求解器自主研发**，C 包采购“结构随机振动分析模块”一套。与采购商业软件授权根本不同，招标文件明确要求：**中标人交付软件模块及可编译该模块的全部源代码，自交付起软件模块及全部代码的知识产权归采购人所有**。这决定了 C 包的本质是“随机振动算法库委托研发 + 源代码与知识产权完整移交”，对投标人的要求不是“有一套能用的软件”，而是：

（1）**算法自主可控**——随机振动全部核心算法（功率谱、非平稳时域、响应谱）具备完整的数学原理依据与自主代码实现，不依赖任何受限的第三方商业库；

（2）**代码可移交、可维护**——源代码工程化、文档化，采购人接手后能够独立编译、修改、扩展与二次开发；

（3）**可嵌入自研框架**——模块接口清晰、数据协议明确，能够嵌入采购人自研的国产结构动力学求解器框架，并支持与国产软硬件平台集成。

### 1.2 C 包技术任务的本质理解

招标文件第三章对 C 包的定位是：“面向国产结构动力学求解器研发，提供结构随机振动功率谱分析算法库、结构随机振动时域分析算法库和响应谱分析算法库。”我方对其技术本质的理解如下：

确定性动力分析回答“给定载荷下结构怎么动”，而**随机振动回答“在统计描述的随机激励下，结构响应的统计特征是什么”**——火箭发射段的宽频随机载荷、路面对车辆的随机激励、地震对结构的随机/反应谱激励，在随机振动与响应谱分析框架下，通常用**功率谱密度**或**反应谱**刻画。C 包要交付的，正是把“随机激励的统计描述”映射为“结构响应的统计描述”的三类算法：

（1）**平稳随机振动（频域）**——已知输入功率谱矩阵 $\mathbf S_{FF}(\omega)$，求响应功率谱 $\mathbf S_{uu}(\omega)$ 与响应标准差 $\sigma_u$；对于零均值随机响应，标准差等于均方根（RMS）。核心是频响函数 $\mathbf H(\omega)$ 与谱关系 $\mathbf S_{uu}=\mathbf H\,\mathbf S_{FF}\,\mathbf H^{\mathsf H}$，其理论基础可参考随机振动与结构动力学经典文献 [7–9]；

（2）**非平稳随机振动（时域）**——输入统计特征随时间变化（如发射、冲击的调制随机过程），求时变标准差 $\sigma_u(t)$、时变均方根 $u_{\mathrm{rms}}(t)$ 与固定时刻的响应边缘概率密度函数 $p_u(x,t)$；

（3）**响应谱分析**——已知设计反应谱、结构模态和阻尼等数据，用模态叠加估计结构峰值响应 $R$（由各阶模态峰值贡献 $r_i$ 组合而成）。

### 1.3 C 包与 A、B 包的关系及在整体求解器中的定位

本项目 A、B、C 三包面向同一个国产结构动力学求解器，共享同一套有限元建模底座（混合网格 → 单元/材料/连接/边界 → $\mathbf M$/$\mathbf C$/$\mathbf K$ 组装），并在数据上层层衔接，构成“模态 → 确定性响应 → 随机响应”的完整动力学分析链：

- **A 包（结构模态分析）**——求结构的固有频率与振型，是整个动力学分析体系的地基；
- **B 包（结构动力响应分析）**——在模态基础上求确定性载荷（瞬态/频响）下的响应，且招标文件明确要求其“支持平稳随机振动频域求解调用和非平稳随机振动时域求解调用，为后续随机振动分析模块提供稳定接口”——即 B 包在设计上即为 C 包预留了求解调用入口；
- **C 包（结构随机振动分析，本投标包）**——处理随机/反应谱激励下的统计响应，位于这条分析链的最上层：其 CQC/SRSS 功率谱与响应谱四种叠加法以**模态数据（A 包能力）**为输入，其 PEM 与非平稳时域分析以**频域/时域求解能力（B 包能力）**为后端。

也就是说，在采购人最终的整体求解器蓝图中，**C 包自然地以 A 包的模态数据与 B 包的求解能力为上游**。我方对这一关系的应对策略分别面向“C 包独立交付验收”与“采购人整体集成”两种情形：

（1）**C 包独立交付验收时**——C 模块提供标准化数据与求解服务接口，可接收质量矩阵、阻尼矩阵、刚度矩阵、模态数据以及频域/时域求解结果；同时配套标准算例、基准数据和必要的测试支撑组件，确保 C 包核心算法可独立运行、验证和验收，不以 A、B 包是否同时交付为前提（详见 1.4）；

（2）**面向整体集成时**——C 模块的数据接口与 A、B 包的自然边界保持一致：模态数据接口承接 A 包输出，频域/时域求解服务接口承接 B 包为随机振动预留的调用能力。具体数据结构、调用协议和集成边界在项目需求确认与接口设计阶段与采购人共同固化，以降低后续适配工作量，避免接口孤岛与重复开发。

这一设计既保证 C 包核心算法作为独立标的可验证、可验收，又使其能够复用采购人现有能力或 A、B 包成果，契合采购人“三模块协同、统一求解器”的总体目标。

### 1.4 C 包支撑能力与接口边界

C 包三类分析算法需要相应的结构动力学基础数据与求解能力：

- 功率谱法的 **CQC / SRSS** 与**响应谱分析**需要固有频率、振型、模态阻尼和模态参与系数等模态数据；
- **PEM（虚拟激励法）**需要调用确定性频率响应求解能力；
- **非平稳随机振动分析**需要调用时域积分求解能力。

基于上述依赖关系，本方案采用“**随机振动算法库 + 标准支撑接口 + 测试支撑组件**”的技术组织方式：

（1）**随机振动算法库**负责实现功率谱分析、非平稳随机振动分析和响应谱分析等 C 包核心功能；

（2）**标准支撑接口**用于接收采购人现有框架或 A、B 模块提供的模型数据、$\mathbf M$/$\mathbf C$/$\mathbf K$ 矩阵、模态数据，以及频域和时域求解服务；

（3）**测试支撑组件**提供标准模型、基准数据、模拟求解服务及必要的轻量化测试能力，用于在 A、B 模块尚未接入时完成 C 包核心算法的单元测试、集成测试和独立验收；正式集成后，C 模块通过同一标准接口复用采购人现有求解器或 A、B 模块提供的基础能力。

对于招标文件要求支持的杆、梁、壳、实体、连接、质量、阻尼、材料和约束等模型能力，本模块通过统一模型适配接口和分析数据协议予以承接，确保相应模型形成的矩阵、模态、载荷及结果恢复数据能够完整进入随机振动分析流程，并可按节点、单元和指定响应分量输出功率谱、均方根、时变统计量及响应谱组合结果。

投标阶段提供初步接口规范，明确矩阵、模态、载荷谱、反应谱、响应位置及结果数据的基本结构和调用方式；合同实施后，根据采购人总体求解器的既有数据结构完成接口确认、适配和版本固化。

该方案既保证 C 包核心算法能够独立验证和验收，又可充分复用 A、B 包或采购人现有求解器的基础能力，避免重复建设完整的有限元建模、模态和动力响应求解系统。

## 二、总体方案

### 2.1 总体技术路线

C 模块采用“**一套标准支撑接口 + 三类算法库 + 一组测试支撑组件**”的总体技术路线：

**一套标准支撑接口**——面向采购人总体求解器定义统一的数据与服务边界，接收模型及载荷数据、$\mathbf M$/$\mathbf C$/$\mathbf K$ 矩阵、模态数据，并调用频率响应和时间积分服务；接口同时支持采购人现有框架、A/B 模块及测试支撑组件接入。

**三类算法库**——对应招标 C 包要求的三类算法：

（1）**功率谱分析算法库**：CQC、SRSS、PEM（虚拟激励法）求响应功率谱，及响应均方根计算；

（2）**随机振动时域分析算法库**：非平稳随机振动的时变均方根与响应概率密度函数计算；

（3）**响应谱分析算法库**：单点/多点激励下 SRSS、NRL、GRP、CQC 四种模态组合的峰值响应 $R$ 估计。

**一组测试支撑组件**——提供标准模型、矩阵与模态基准数据、频域/时域模拟服务、算例驱动和结果比对工具，用于脱离 A、B 模块开展算法测试、回归验证和独立验收。

技术路线的关键决策：

**（1）标准化模态数据作为枢纽。** CQC/SRSS 功率谱和响应谱四种叠加法共享统一的模态数据结构（频率、振型、模态质量/刚度/阻尼、模态参与系数），支持 A 包、采购人现有模态模块或测试基准数据接入，实现一次计算、多分析复用。

**（2）频响服务与 PEM 深度结合。** PEM 将随机响应谱计算转化为一系列确定性简谐响应计算。C 模块通过统一频响服务接口批量提交虚拟激励并获取响应，使其能够复用 B 包或采购人现有频率响应能力；独立测试时则由测试支撑组件提供基准频响数据或轻量化验证服务。

**（3）面向大规模数据的接口设计。** 矩阵和谱数据接口支持稀疏存储、批量频点、分块输出及按需提取，避免不必要的数据复制；具体模态、频响和时域求解算法由所接入的 A/B 模块或采购人求解服务负责。

**（4）多工况组织。** 激励谱、反应谱、约束集合以“工况（Case）”对象组织，支持单次建模、多工况批量求解与分组导出，满足招标对多工况计算组织的要求。

### 2.2 软件总体架构设计

模块采用自下而上四层架构，各层接口明确、可独立测试：

**Layer 0 —— 数据协议与适配层。**

（1）定义模型标识、自由度映射、矩阵、模态、随机激励谱、反应谱、工况及结果数据协议，并提供输入完整性、维度一致性、单位和频率网格校验；

（2）通过适配器与采购人现有模型和结果数据格式衔接。投标阶段提供初步数据协议和接口定义；合同实施后结合采购人既有数据结构完成字段映射、格式适配和接口版本固化，避免将 C 模块绑定于单一前后处理格式。

**Layer 1 —— 模型适配与分析数据准备层。** 承接杆/梁、二维剪切单元与壳、实体、刚性及物理连接、质量、阻尼、材料和约束等模型信息，建立其与随机振动分析所需矩阵、模态、响应位置及载荷通道之间的映射，并维护节点、单元和响应分量对应的结果恢复关系。基础运动激励通过强制位移/基础运动输入通道传递至随机振动和响应谱分析。

**Layer 2 —— 支撑求解服务接口层。** 定义模态数据获取、频率响应批量求解和时域积分调用接口，可接入 A 包、B 包或采购人现有求解器；测试支撑组件按照同一接口提供基准数据和轻量化验证服务，以保证接口可独立测试。

**Layer 3 —— 随机振动算法与应用接口层。** 实现 2.1 节三类算法库，组织分析工况、算法参数和结果提取；对外提供稳定的库调用接口及测试驱动，支持框架集成、独立算例验证和批量回归测试。

### 2.3 与采购人自研求解器框架的集成方案

为保证“接口清晰、可嵌入自研国产求解器框架”，我方承诺：

（1）**纯库交付**：源代码 + CMake 构建脚本交付，编译产物为静态/动态库，不捆绑 GUI、不依赖商业组件；第三方依赖仅限可再分发的开源数值库，随源码一并交付并锁定版本；

（2）**两类调用方式**：数据注入方式——采购人框架直接提供 $\mathbf M$/$\mathbf C$/$\mathbf K$、模态及载荷谱等数据；服务调用方式——C 模块通过标准接口调用采购人或 A/B 模块提供的模态、频响和时域求解服务；

（3）**数据协议文档化**：投标阶段提供初步接口规范；交付时形成经联调确认并完成版本固化的正式接口文档和示例代码，覆盖矩阵格式、模型与自由度映射、模态数据结构、求解服务接口、激励谱/反应谱、结果恢复及结果数据结构；

（4）**联调支持**：交付期内安排专人与采购人研发团队按指定编译环境（含国产软硬件平台）完成构建与联调验证。

### 2.4 质量保证与验证体系

（1）**三级算例验证**：解析解算例（单自由度系统白噪声响应等有闭式解的问题）→ 标准基准算例 → 工程规模算例（复杂混合网格、多点激励），每个 ▲ 算法条款至少覆盖前两级；

（2）**交叉校核**：CQC 与 PEM 在同一算例上互校，频域平稳极限与时域非平稳长时解互校，蒙特卡洛抽样作为随机响应统计量的参考解；

（3）**自动回归**：全部算例纳入自动化回归测试集，每次代码变更后全量运行；

（4）**文档交付**：算法原理手册、用户手册（中文）、接口文档、算例报告随软件一并交付。

### 2.5 小结

总体方案以“标准支撑接口 + 标准化模态数据 + PEM/频响服务结合”组织 C 包随机振动三类算法，以“算法库交付 + 数据/服务两类调用方式 + 数据协议文档化”保证可嵌入性，以“测试支撑组件 + 三级算例 + 交叉校核 + 自动回归”保证独立验证和质量。

## 三、功能要求响应与实现方案

本章严格对应招标文件第三章 C 包“结构随机振动分析模块”技术参数，对 3 个 ★ 功能逐条响应、对 20 个 ▲ 性能指标逐条“完全响应”。全部 ★/▲ 项承诺**无负偏离**；招标文件明确要求证明材料的条款，按要求提供算法原理描述与算例测试报告。

### 3.1 ★ 核心功能逐条响应（3 项，全部完全满足）

**★1 具备结构平稳随机振动分析功能。** 响应：**完全满足，无偏离**。面向线性结构的平稳随机振动分析，以输入激励功率谱矩阵

$$
\mathbf S_{FF}(\omega)
=
\left[
S_{F_aF_b}(\omega)
\right]
$$

描述已映射至结构物理自由度空间的各输入载荷分量的谱强度及相关关系。其中，$S_{F_aF_b}(\omega)$ 表示结构物理自由度上的输入载荷分量 $a$ 与 $b$ 在频率 $\omega$ 处的互功率谱，用于描述两载荷分量的频域相关强度和相位关系；当 $a=b$ 时为自功率谱。通过频响函数

$$
\mathbf H(\omega)
=
\left(
\mathbf K-\omega^2\mathbf M+i\omega\mathbf C
\right)^{-1}
$$

计算响应功率谱矩阵

$$
\mathbf S_{uu}(\omega)
=
\mathbf H(\omega)\,
\mathbf S_{FF}(\omega)\,
\mathbf H^{\mathsf H}(\omega)
$$

#### （二）响应功率谱算法（▲10–▲13，完全响应）

**▲10 支持 CQC 算法求解响应功率谱。** 完全响应。在线性、经典阻尼、实模态及平稳随机激励条件下，采用模态叠加计算响应功率谱，并保留截断模态集合内的全部模态自谱项和交叉谱项。本节中的 $\mathbf S_{FF}(\omega)$ 均指已映射至结构物理自由度空间的输入激励功率谱矩阵。设第 $i$ 阶模态频响 $H_i(\omega)=\big(\omega_i^2-\omega^2+2i\xi_i\omega_i\omega\big)^{-1}$（单位模态质量归一化），则模态（广义）力互谱为

$$
S_{f_i f_j}(\omega)
=
\boldsymbol{\phi}_i^{\mathsf T}
\mathbf S_{FF}(\omega)
\boldsymbol{\phi}_j
$$

指定自由度处的响应自功率谱为

$$
S_{u_pu_p}(\omega)
=
\sum_i\sum_j
\phi_{pi}\phi_{pj}
H_i(\omega)H_j^*(\omega)
S_{f_i f_j}(\omega)
$$

其中，$\boldsymbol{\phi}_i$ 为第 $i$ 阶振型向量，$\phi_{pi}$ 为其在指定响应自由度 $p$ 处的分量。该方法考虑模态间交叉相关影响，适用于密集模态结构；计算精度同时取决于模态截断范围、阻尼模型、频率离散及输入谱数据。此处 CQC 指功率谱分析中的完全二次模态谱组合，不同于 ▲20 响应谱分析中采用相关系数 $\rho_{ij}$ 组合各阶峰值响应的 CQC 方法。

**▲11 支持 SRSS 算法求解响应功率谱。** 完全响应。在 ▲10 模态叠加基础上**忽略模态交叉项（仅保留 $i=j$ 项）**：

$$
S_{u_pu_p}(\omega)
=
\sum_i
\phi_{pi}^2
\left|H_i(\omega)\right|^2
S_{f_i f_i}(\omega)
$$

适用于固有频率分离较好、模态响应相关性较弱的结构，计算高效；在上述条件下，其结果与 CQC 结果趋于一致。

**▲12 支持 PEM 法求解响应功率谱算法。**（提供算法原理描述与算例测试报告）完全响应。**虚拟激励法（Pseudo Excitation Method）** 由林家浩等提出并系统发展，是随机振动分析中的重要高效算法 [1–3]。该方法将 Hermitian 半正定的激励功率谱矩阵逐频点采用特征值分解或带主元 Cholesky 等稳健方法分解为 $\mathbf S_{FF}(\omega)=\sum_k \mathbf s_k(\omega)\,\mathbf s_k^{\mathsf H}(\omega)$。对每个已定义在结构物理自由度空间的谱分量构造确定性虚拟简谐激励 $\tilde{\mathbf F}_k(\omega,t)=\mathbf s_k(\omega)\,e^{i\omega t}$，求其确定性简谐响应 $\tilde{\mathbf u}_k(\omega)=\mathbf H(\omega)\,\mathbf s_k(\omega)$，则响应功率谱

$$
\mathbf S_{uu}(\omega)
=
\sum_k
\tilde{\mathbf u}_k(\omega)\,
\tilde{\mathbf u}_k^{\mathsf H}(\omega)
$$

在所采用频响模型、谱分解精度和模态截断范围内，PEM 保留输入通道相关性及相应的模态交叉贡献，将随机响应谱计算转化为一组确定性频响计算。所需确定性频响计算次数等于保留的有效谱分量数；当输入谱矩阵低秩或有效激励通道数明显少于保留模态数时，可较完整模态交叉项展开显著降低计算量。提供算法原理描述与算例测试报告，并与 CQC 或解析解进行交叉校核。

**▲13 支持响应均方根计算。**（提供算例测试报告）完全响应。采用“响应功率谱沿双边角频率积分直接得到方差”的谱归一化约定，指定响应分量的方差为

$$
\sigma_u^2
=
\int_{-\infty}^{+\infty}
S_{uu}(\omega)\,
\mathrm d\omega
$$

响应均方根为 $u_{\mathrm{rms}}=\sqrt{\mu_u^2+\sigma_u^2}$；对于通常采用的零均值随机响应，$u_{\mathrm{rms}}=\sigma_u$。若采用其他傅里叶变换或单边谱约定，则同步调整积分区间和归一化系数。输出节点/单元指定响应量的均方根云图数据；在共振峰附近进行自适应频率网格加密以提高积分分辨率，并依据积分收敛准则控制数值误差。提供算例测试报告（采用具有物理质量 $m$、固有频率 $\omega_n$、阻尼比 $\xi$ 的单自由度系统作为闭式校核基准，与 ▲10–▲12 模态叠加所采用的单位模态质量归一化频响相互独立；在力功率谱 $S_0$（双边白噪声）激励下，其位移方差解析解为 $\sigma_u^2=\dfrac{\pi S_0}{2\xi\omega_n^3 m^2}$）。

#### （三）非平稳随机振动算法（▲14–▲15，完全响应）

**▲14 非平稳随机振动分析支持时变响应均方根计算。**（提供算例测试报告）完全响应。非平稳激励用**演变功率谱** $\mathbf G_{FF}(\omega,t)$ 描述 [10]（区别于平稳谱 $\mathbf S_{FF}(\omega)$）。一种常用的标量调制模型为 $\mathbf G_{FF}(\omega,t)=|A(\omega,t)|^2\,\mathbf S_{FF}^0(\omega)$，其中 $\mathbf S_{FF}^0$ 为基准平稳谱、$A(\omega,t)$ 为时变调制（包络）函数；该模型用于描述调制型非平稳过程，不代表所有一般非平稳过程。采用**时域虚拟激励法** [3]：将基准谱逐频点分解为 $\mathbf S_{FF}^0(\omega)=\sum_k\mathbf s_k(\omega)\,\mathbf s_k^{\mathsf H}(\omega)$，构造时变虚拟激励 $\tilde{\mathbf F}_k(\omega,t)=A(\omega,t)\,\mathbf s_k(\omega)\,e^{i\omega t}$，通过标准时域求解服务接口逐步求解 $\tilde{\mathbf u}_k(\omega,t)$（因结构存在记忆效应，不能将平稳公式逐时刻照搬），重构输出演变功率谱 $\mathbf G_{uu}(\omega,t)=\sum_k\tilde{\mathbf u}_k\tilde{\mathbf u}_k^{\mathsf H}$。采用“演变功率谱沿双边角频率积分直接得到时变方差”的谱归一化约定，则

$$
\begin{aligned}
\sigma_u^2(t)
&=
\int_{-\infty}^{+\infty}
G_{uu}(\omega,t)\,
\mathrm d\omega,\\
\sigma_u(t)
&=
\sqrt{\sigma_u^2(t)},\\
u_{\mathrm{rms}}(t)
&=
\sqrt{\mu_u^2(t)+\sigma_u^2(t)}.
\end{aligned}
$$

零均值时有 $u_{\mathrm{rms}}(t)=\sigma_u(t)$。若采用单边谱或其他傅里叶变换约定，则同步调整积分区间和归一化系数。提供算例测试报告，测试中采用平稳极限解和蒙特卡洛参考结果进行交叉校核。

**▲15 非平稳随机振动分析支持响应概率密度函数的计算。**（提供算例测试报告）完全响应。对于高斯激励且初始条件为确定性或高斯的线性系统，响应在每个固定时刻仍为高斯分布，其边缘概率密度由 $\mu_u(t)$、$\sigma_u(t)$ 直接派生（零均值时 $\mu_u=0$）：

$$
p_u(x,t)
=
\frac{1}{\sqrt{2\pi}\,\sigma_u(t)}
\exp\!\left[
-
\frac{(x-\mu_u(t))^2}{2\sigma_u^2(t)}
\right]
$$

此为高斯闭式后处理，无需另解动力学方程。对于线性系统下的非高斯激励，在已给定输入概率模型、样本数据或样本生成规则的条件下，可通过激励样本系综生成、批量线性时域求解和核密度估计计算 $p_u(x,t)$；仅有功率谱不足以唯一确定非高斯过程的概率分布。需注意：固定时刻的边缘密度不等于整个时段最大值的分布，峰值/越界概率另需极值理论。提供算例测试报告。

#### （四）响应谱分析算法（▲16–▲20，完全响应）

先由模态 $(\omega_i,\boldsymbol{\phi}_i)$、阻尼比 $\xi_i$、质量阵 $\mathbf M$ 与地震输入方向向量 $\mathbf r$ 计算模态参与系数

$$
\Gamma_i
=
\frac{
\boldsymbol{\phi}_i^{\mathsf T}
\mathbf M
\mathbf r
}{
\boldsymbol{\phi}_i^{\mathsf T}
\mathbf M
\boldsymbol{\phi}_i
}
$$

此为通用归一化下的表达式；若振型按单位模态质量归一化，则分母 $\boldsymbol{\phi}_i^{\mathsf T}\mathbf M\boldsymbol{\phi}_i=1$，退化为 $\Gamma_i=\boldsymbol{\phi}_i^{\mathsf T}\mathbf M\mathbf r$，与 ▲10 频响所采用的归一化约定一致。

以位移反应谱为例，第 $i$ 阶模态对指定自由度 $p$ 的位移峰值贡献为

$$
r_{pi}^{(u)}
=
\Gamma_i\,
\phi_{pi}\,
S_d(\omega_i,\xi_i)
$$

其中，$S_d(\omega_i,\xi_i)$ 为位移反应谱在第 $i$ 阶固有频率和阻尼比处的纵坐标，$\phi_{pi}$ 为第 $i$ 阶振型在自由度 $p$ 处的分量。目标为加速度、内力或应力时，采用相应反应谱量并通过对应结果恢复算子计算，不能直接套用位移峰值公式。对同一指定响应量，将各阶峰值贡献简记为 $r_i$，再按以下方法组合得到总峰值估计 $R$。由于各阶峰值通常不在同一时刻发生，组合结果一般不保留正负号和峰值发生时刻。

**▲16 响应谱分析支持单点、多点激励施加。** 完全响应。支持单一支座统一激励，以及多支座分组、各支座反应谱和输入方向的定义。多点或多支座输入涉及不同支座运动的空间相关性及支座贡献组合，具体规则依据选定规范配置，不直接套用单点、单方向激励的模态参与系数公式。

**▲17 支持 SRSS 叠加法。**（提供算法原理描述与算例测试报告）完全响应。对于同一指定响应量，令 $r_i$ 为第 $i$ 阶模态的峰值贡献，则组合后的总峰值估计为 $R=\sqrt{\sum_i r_i^2}$。该方法适用于固有频率分离较好、各阶峰值响应相关性较弱的情形；相关模态组合理论可参考文献 [4–6]。

**▲18 支持 NRL 叠加法。**（提供算法原理描述与算例测试报告）完全响应。NRL（美国海军研究实验室法，Naval Research Laboratory）取绝对值最大的模态贡献与其余模态的 SRSS 结果相加 [11]。令 $k=\arg\max_i|r_i|$，则

$$
R_{\mathrm{NRL}}
=
|r_k|
+
\sqrt{
\sum_{i\ne k}r_i^2
}
$$

该方法属于工程中使用的保守组合方法。

**▲19 支持 GRP 叠加法。**（提供算法原理描述与算例测试报告）完全响应。GRP（分组法，Grouping Method）将频率相近的模态按选定规范或频率间隔阈值进行分组 [6]，组内采用模态贡献绝对值相加，组间采用 SRSS 组合，以考虑密集模态影响；工程实现中明确并记录所采用的分组准则和阈值。

**▲20 支持 CQC 叠加法。**（提供算法原理描述与算例测试报告）完全响应。基于模态相关系数 $\rho_{ij}$ 的完全二次组合 [4,5] $R=\sqrt{\sum_i\sum_j \rho_{ij}\, r_i r_j}$。在线性结构、经典阻尼及白噪声输入假设下，令频率比 $\beta_{ij}=\omega_j/\omega_i$，一种常见的相关系数形式为：

$$
\rho_{ij}
=
\frac{
8\sqrt{\xi_i\xi_j}\,
(\xi_i+\beta_{ij}\xi_j)\,
\beta_{ij}^{3/2}
}{
(1-\beta_{ij}^2)^2
+
4\xi_i\xi_j\beta_{ij}(1+\beta_{ij}^2)
+
4(\xi_i^2+\xi_j^2)\beta_{ij}^2
}
$$

该方法用于考虑密集模态之间的相关性。工程实现中明确所采用的规范版本、频率比定义、阻尼模型和适用假设；采用其他规范或输入假设时，按相应规定选取相关系数。

（1）**形态适配**：以随机振动算法库为核心交付物，通过标准支撑接口接收采购人已有的矩阵、模态及求解服务；测试支撑组件可用于脱离 A、B 模块开展独立算例验证；

（2）**代码适配**：全部源代码以工程化标准交付（统一规范、完整注释、CMake 构建、自动化测试），保证采购人接手后可独立编译、修改、扩展，与“知识产权完整移交”的采购本质一致；

（3）**生态适配**：模型和结果格式采用适配器机制衔接采购人现有前后处理工具链，避免将随机振动算法绑定于单一文件格式，降低对现有建模和数据处理流程的影响。

### 4.2 与三模块体系的协同适用性

采购人 A、B、C 三包的最终目标是一套协同工作的统一国产求解器（三包共享有限元底座、模态与求解数据层层衔接，详见 1.3）。本 C 包方案在“体系适配”维度具备如下适用性：

（1）**接口前向兼容**：C 模块的模态数据结构、频域/时域求解调用接口按 A、B 包的自然边界定义，并通过版本化接口协议和适配器降低后续模块接入的改造工作量；

（2）**集成风险可控**：C 模块配套标准算例、基准数据和测试支撑组件，核心算法可在 A、B 模块尚未接入时独立验证；整体集成时再切换至正式求解服务，降低跨包进度和接口变化带来的风险；

（3）**演进路径平滑**：分层架构使测试支撑组件、A/B 包模块和采购人自研实现遵循同一服务接口，上层随机振动算法无需因底层求解服务替换而整体重构。

### 4.3 工程模型适用性

（1）**混合网格**：模型适配接口覆盖 ▲1–▲9 所列 1D/2D/3D 单元、连接、质量、阻尼、材料和约束类型，保持模型数据到矩阵、模态、响应位置、载荷通道及结果恢复数据的映射关系，支持按节点、单元和指定响应分量提取随机振动分析结果；

（2）**大规模问题**：接口支持稀疏矩阵、批量频点、谱矩阵分解及频响/时域服务复用，结合 PEM 减少显式处理大量模态交叉项的开销，支撑工程规模模型的随机振动分析；

（3）**复杂激励**：支持单点/多点激励、多支座反应谱及经 SPCD 或等效通道输入的基础运动随机激励；PEM 通过输入谱矩阵分解保留多点激励的相关信息；

（4）**平稳与非平稳分析贯通**：在相应适用假设下，以一套模块实现频域平稳分析（功率谱/RMS）和时域非平稳分析（时变 RMS/概率密度），覆盖招标文件要求的两类随机振动分析功能。

### 4.4 平台与环境适用性

（1）标准 C/C++ 实现，不依赖特定指令集与商业运行时，支持 Linux/Windows 及国产软硬件平台（国产 CPU + 国产操作系统）编译运行，满足“非进口产品”与自主可控要求；

（2）采用 CMake 构建系统，并通过持续集成和目标平台编译测试验证交叉编译与平台移植路径；

（3）Python 绑定便于采购人科研人员快速验证算法与二次开发，匹配高校“研发 + 教学 + 工程服务”的多重场景。

### 4.5 验收与可验证性适用性

招标文件明确要求算例测试报告的 ▲ 条款，其标准算例同时作为验收测试用例；其他算法功能亦纳入内部回归测试集。验收时可在采购人环境下复跑相应算例，通过预先约定的误差指标和通过准则评价结果，使“检验试验根据招标文件项目需求及国家有关标准进行验收”具备可操作依据。


## 五、技术先进性说明

### 5.1 算法体系的先进性

**（1）虚拟激励法（PEM）—— 随机振动的高效算法。** PEM 通过谱分解将随机响应谱计算转化为确定性简谐计算序列 [1–3]，能够在输入谱分解中保留多点激励的相关信息。本方案拟以 PEM 作为功率谱分析的核心算法，并采用时域虚拟激励与逐步积分相结合的技术路线处理非平稳随机振动问题。相关算法按照“数学模型明确、接口边界清晰、标准算例验证、工程算例复核”的方式实施。

**（2）功率谱算法体系覆盖不同适用场景。** CQC 用于考虑模态交叉相关影响，SRSS 适用于模态响应相关性较弱的情形，PEM 适用于通过确定性频响计算处理单点或多点相关激励；三种方法可在适用条件重叠的算例中开展交叉校核 [1–5]。

**（3）响应谱叠加方法类型完整。** SRSS、NRL、GRP、CQC 四种组合方法分别提供忽略模态相关、保守组合、密频分组和相关系数组合等不同处理路径；具体适用范围、分组准则和相关系数按照项目选定规范及分析假设配置 [4–6,11]。

**（4）非平稳随机振动的工程化技术路线。** 采用演变功率谱、时域虚拟激励与逐步积分相结合的方案 [3,10]，面向时变均方根和响应概率密度等招标指标开展实现与算例验证。

**（5）支撑接口具有开放性。** 模态、频率响应和时域积分能力均通过标准服务接口调用，可接入采购人现有求解器、A/B 包模块或测试支撑组件，使随机振动算法与具体底层求解实现解耦，便于后续替换、扩展和性能优化。

### 5.2 软件架构的先进性

（1）**分层解耦**：数据协议、模型适配、求解服务接口和随机振动算法四层边界清晰，算法与底层模型、模态及动力响应求解实现解耦，便于独立替换和升级；

（2）**模态数据枢纽**：CQC/SRSS/响应谱共享标准化模态数据，可复用 A 包或采购人已有模态结果，减少重复计算并保证不同分析之间的数据一致性；

（3）**自动化质量体系**：全部算例纳入回归测试、文档与代码同步交付，按可持续维护的工业软件标准建设，与“源代码移交后由采购人长期演进”的项目定位匹配。

### 5.3 先进性的可落地性

本方案的可落地性建立在我方既有的有限元方法、数值线性代数、大规模科学计算和工程软件开发基础之上。现有证明材料能够支撑团队在有限元离散、矩阵计算、频域问题求解、软件架构设计、自动化测试及国产平台适配等方面的研发能力。

针对本项目新增的 PEM 功率谱分析、非平稳随机振动及响应谱组合算法，我方将按照“原理设计—接口实现—标准算例—交叉校核—工程验证”的路径完成研发和验证：

（1）依据随机振动理论完成算法公式、适用假设、输入输出和数值流程设计；

（2）通过标准支撑接口复用模态、频率响应和时域积分能力，实现随机振动算法与底层求解器解耦；

（3）采用解析解、标准基准算例和工程模型逐级验证，并通过 CQC、SRSS、PEM、蒙特卡洛参考解等方式开展交叉校核；

（4）将全部算例纳入自动化回归测试，形成可重复执行的验收测试集；

（5）同步交付算法原理、接口说明、算例报告和源代码，保证成果可编译、可验证、可维护。

因此，本方案所述先进性体现为技术路线和工程实施体系的先进性。相关功能是否达到预期，以最终代码、算例测试报告及采购人环境下的验收结果为准。


### 5.4 后续扩展接口预留

本方案首交范围以招标文件 3 个 ★ 功能和 20 个 ▲ 指标为边界。分层解耦架构和标准支撑接口为后续能力扩展预留了条件，但本节内容仅用于说明架构可扩展性，不作为首交范围的功能承诺。

后续可扩展方向包括：在已有样本统计和概率密度计算接口基础上扩展非高斯非平稳随机过程建模 [12]；在频繁参数扫描、可靠性分析和敏感性分析等场景中，引入多项式混沌展开、稀疏概率配置等不确定性量化代理模型 [13,14]；在更复杂结构场景中扩展非经典阻尼、复模态随机响应、越界概率和概率风险分析等后处理能力。

上述方向均应通过新增算法模块、扩展数据接口或增加后处理服务的方式逐步纳入，不影响首交版本对招标指标的实现。

### 5.5 小结

本方案的先进性体现为：**算法上**以 PEM 为核心、功率谱/非平稳/响应谱算法谱系完整；**架构上**采用标准支撑接口、模态数据枢纽和分层解耦组织，可复用采购人或 A/B 包的基础能力；**工程上**通过测试支撑组件、自动回归以及“算法 + 原理文档 + 标准算例”实现可验证交付。三者共同保证 C 包满足招标技术参数，并形成便于集成和持续演进的随机振动算法资产。

本说明针对 C 包（结构随机振动分析模块），从项目理解、总体方案、功能要求逐条响应、方案适用性与技术先进性五个方面完整阐述了我方技术响应。我方承诺：**3 个 ★ 关键功能全部无偏离满足，20 个 ▲ 重要指标全部“完全响应”，并对招标文件明确要求的条款提供算法原理描述和算例测试报告等证明材料**，合同签订后 2 个月内完成交付，源代码与知识产权完整移交，并以 1 年质保与 7×24 响应保障采购人使用。
