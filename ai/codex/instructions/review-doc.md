# Review Doc Workflow

用户要求“技术审校”“review-doc”“审一下技术正确性”等任务时使用本流程。

## 目标

对结构动力学文档做技术正确性深审。

## 范围判定

- 如果用户给出文件路径，就审查该路径。
- 如果用户未给出路径，则审查本次会话中改动过的 `docs/*.md`。
- 如果本次会话无改动，则审查两篇主文档：
  - [docs/三包知识框架.md](../../../docs/三包知识框架.md)
  - [docs/procurement/总体方案及功能要求、方案适用性及技术先进性说明.md](../../../docs/procurement/总体方案及功能要求、方案适用性及技术先进性说明.md)

## 执行方式

按 [dynamics-reviewer.md](dynamics-reviewer.md) 的清单执行审校。输出时致命问题优先，每条带 `文件:行号`，并给出建议修复方案。

除非用户明确要求直接修改，否则先不要改文件，等用户确认。
