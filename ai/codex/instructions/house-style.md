# House Style

撰写、修改、续写或审阅 `docs/` 下结构动力学文档（知识框架、投标方案、技术需求、招标解读）时使用本说明。

基础硬规则见 [../AGENTS.md](../AGENTS.md) 的“写作约定”。本文件补充语气视角、章节结构、符号约定、术语桥用法与改动纪律等隐性手感。

## 语气与视角

- 从有限元流程出发讲知识：先建立“建模 -> 组装 -> 求解”的熟悉锚点，再引入新概念。
- 解释“为什么”而非只罗列“是什么”：每个算法给出核心直觉，如“复杂振动 = 基本振动方式的叠加”。
- 工程价值用真实案例支撑（塔科马大桥、航天随机振动鉴定、车辆路面谱），但案例的物理解释必须准确。
- 行文凝练、有判断力，避免空话与营销腔；投标方案可有“我方”主语，知识框架不用。

## 章节结构

- 知识框架：`定义 -> 核心要点 -> 术语桥 -> 引言 -> 原理与机制（四层次：几何/机理/数学/计算）-> A/B/C 逐包展开`。新增小节挂到对应层次或包下。
- 投标方案：按招标“格式自拟”的层级（一、项目理解；二、总体方案；三、逐条响应……）。每个 ▲ 项按“算法实现 + 原理描述 + 算例测试报告/截图”三位一体组织。
- 用 `## ✦` / `###` 层级与现有文档保持一致；适当用表格做“术语/卡片 ↔ 机理”对照。

## 符号约定

- 矩阵/向量粗体：$\mathbf M$、$\mathbf K$、$\mathbf C$、$\mathbf u$、$\hat{\mathbf F}$；振型 $\boldsymbol\phi$、$\boldsymbol\Phi$；特征值 $\lambda$、频率 $\omega$。
- 沿用既有写法：动刚度 $\mathbf K-\omega^2\mathbf M+i\omega\mathbf C$，频响 $\mathbf H(\omega)$，谱关系 $\mathbf S_{uu}=\mathbf H\,\mathbf S_{FF}\,\mathbf H^{\mathsf H}$。
- 引入新符号必须就近说明，避免与既有符号冲突，尤其 $\sigma_s$（谱移位，量纲为频率平方）与 $\sigma_u$（响应标准差）无关，勿混用。

## 术语桥

以 [../../docs/三包知识框架.md](../../../docs/三包知识框架.md) 的“术语桥”表为准，建立力学术语、数学/通俗表述、NASTRAN 卡片名三方对应：

- NASTRAN 卡片名（MAT1/RBE2/SPCD/MPC 等）一律按其力学机理理解和书写，写标书时再做“卡片名 ↔ 机理”对应。
- 不另立同义术语；同一概念全文用词统一。新术语先进术语桥再使用。

## 改动纪律

- 知识框架只讲知识，招标条款、评分、投包策略一律引到采购文档，不在知识框架展开。
- 不确定的物理/数学结论，宁可标注存疑也不要写成定论；必要时按 [dynamics-reviewer.md](dynamics-reviewer.md) 复核，术语不一致按 [terminology-guardian.md](terminology-guardian.md) 巡检。
