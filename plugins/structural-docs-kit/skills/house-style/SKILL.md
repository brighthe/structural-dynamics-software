# 家族写作风格（house-style）

撰写或修改 `docs/` 下文档时遵循本风格，使新内容与现有文档浑然一体。

> 基础硬规则（全中文正文、LaTeX `$`/`$$` 成对、★/▲ 含义与响应原则、交叉引用用相对路径链接）见 `CLAUDE.md`「写作约定」。本手册只补 CLAUDE.md 说不清的**隐性手感**。

## 1. 语气与视角

- **从有限元流程出发**讲知识：先建立"建模 → 组装 → 求解"的熟悉锚点，再引入新概念。
- 解释"为什么"而非只罗列"是什么"：每个算法给出**核心直觉**。
- 工程价值用真实案例支撑（塔科马大桥、航天随机振动鉴定、车辆路面谱），物理解释必须准确。
- 行文凝练、有判断力，避免空话与营销腔；投标方案可有"我方"主语，知识框架不用。

## 2. 章节结构

- 知识框架：`定义 → 核心要点 → 术语桥 → 引言 → 原理与机制（四层次：几何/机理/数学/计算）→ A/B/C 逐包展开`。
- 投标方案：按招标"格式自拟"层级（一、项目理解；二、总体方案；三、逐条响应…），每个 ▲ 项按"算法实现 + 原理描述 + 算例测试报告/截图"三位一体组织。
- 用 `## ✦` / `###` 层级与现有文档一致；适当用表格做"术语/卡片 ↔ 机理"对照。

## 3. 符号约定（沿用现有文档）

- 矩阵/向量粗体：$\mathbf M$、$\mathbf K$、$\mathbf C$、$\mathbf u$、$\hat{\mathbf F}$；振型 $\boldsymbol\phi$、$\boldsymbol\Phi$；特征值 $\lambda$、频率 $\omega$。
- 沿用既有写法：动刚度 $\mathbf K-\omega^2\mathbf M+i\omega\mathbf C$，频响 $\mathbf H(\omega)$，谱关系 $\mathbf S_{uu}=\mathbf H\,\mathbf S_{FF}\,\mathbf H^{\mathsf H}$。
- 引入新符号必须就近说明，避免冲突——尤其 $\sigma_s$（谱移位，量纲为频率平方）与 $\sigma_u$（响应标准差）无关，勿混用。

## 4. 术语桥（唯一权威）

以 `docs/三包知识框架.md` 的"术语桥"表为准，建立**力学术语 ↔ 数学/通俗 ↔ NASTRAN 卡片名**三方对应：NASTRAN 卡片名一律按力学机理理解书写，不另立同义术语，新术语先进术语桥再使用。

## 5. 改动纪律

- 知识框架**只讲知识**，招标条款/评分/投包策略一律引到采购文档，不在知识框架展开。
- 不确定的物理/数学结论，宁可标注存疑也不写成定论；必要时建议调用 `dynamics-reviewer` 复核，术语不一致调用 `terminology-guardian`。
